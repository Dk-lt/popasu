Веб-сервис предназначен для управления материально-техническим обеспечением учебного процесса в образовательной организации.
Серверная часть реализована на **ASP.NET Core Web API**, клиентская часть — на **Blazor WebAssembly**, в качестве системы управления базами данных используется **PostgreSQL**.

Приложение полностью разворачивается в **Docker-контейнерах** и управляется с помощью **Docker Compose**, что обеспечивает изоляцию среды и простоту запуска.

---

## Системные требования

Для запуска проекта необходим только следующий инструмент:

* **Docker Desktop** (включает Docker Engine и Docker Compose)

Установка **.NET SDK** и **PostgreSQL** на локальную машину не требуется.

---

## Запуск приложения

Все команды выполняются из корневой директории проекта, в которой расположен файл
`docker-compose.yml`.

### 1. Клонирование репозитория

```bash
git clone <URL_репозитория>
cd popasu
```

### 2. Сборка и запуск контейнеров

```bash
docker compose up --build
```

При первом запуске Docker автоматически:

* загрузит необходимые образы,
* выполнит сборку серверной и клиентской частей,
* развернёт контейнер с PostgreSQL,
* применит миграции базы данных.

---

## Доступ к компонентам системы

После успешного запуска сервисы будут доступны по следующим адресам:

| Компонент                             | Адрес                                                          |
| ------------------------------------- | -------------------------------------------------------------- |
| Клиентская часть (Blazor WebAssembly) | [http://localhost:5001](http://localhost:5001)                 |
| Серверное API                         | [http://localhost:5000](http://localhost:5000)                 |
| Swagger (документация API)            | [http://localhost:5000/swagger](http://localhost:5000/swagger) |
| PostgreSQL                            | localhost:5432                                                 |

### Параметры подключения к базе данных

* **Имя базы данных:** `popasu`
* **Пользователь:** `postgres`
* **Пароль:** `postgres`

## Остановка работы приложения

Для завершения работы всех контейнеров выполните команду:

```bash
docker compose down
```

Для полной остановки с удалением томов и данных базы данных:

```bash
docker compose down -v
```

---

## Повторная сборка проекта

При внесении изменений в исходный код выполните:

```bash
docker compose build
docker compose up
```

Либо используйте одну команду:

```bash
docker compose up --build
```
