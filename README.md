Веб-приложение для управления материально-техническим обеспечением учебного процесса в образовательной организации с использованием ASP.NET Core Web API в качестве серверной части, Blazor WebAssembly в качестве клиентской части и PostgreSQL в роли системы управления базами данных.  
Проект полностью разворачивается в Docker-контейнерах с использованием Docker Compose.

---

## Структура проекта

```
popasu/
├── popasu.sln
├── docker-compose.yml
├── popasu.Api/
│   └── Dockerfile
├── Client/
│   ├── Dockerfile
│   └── nginx.conf
```

---

## Требования

Для запуска проекта необходимо наличие следующих инструментов:

- Docker Desktop (включает Docker Engine и Docker Compose)

Установка .NET SDK и PostgreSQL на локальную машину не требуется.

---

## Запуск проекта

Все команды выполняются из корневой директории проекта, содержащей файл
`docker-compose.yml`.

### 1. Клонирование репозитория

```bash
git clone <URL_репозитория>
cd popasu
```

### 2. Сборка и запуск контейнеров

```bash
docker compose up --build
```

При первом запуске Docker автоматически:
- загрузит необходимые образы,
- соберёт серверную и клиентскую части приложения,
- запустит базу данных PostgreSQL,
- применит миграции базы данных.

---

## Доступ к компонентам системы

После успешного запуска приложение будет доступно по следующим адресам:

| Компонент | Адрес |
|----------|-------|
| Клиентское приложение (Blazor WebAssembly) | http://localhost:5001 |
| API | http://localhost:5000 |
| Swagger (документация API) | http://localhost:5000/swagger |
| PostgreSQL | localhost:5432 |

### Параметры подключения к базе данных

- Database: `popasu`
- Username: `postgres`
- Password: `postgres`

---

## Остановка приложения

Для остановки всех контейнеров выполните:

```bash
docker compose down
```

Для остановки контейнеров с удалением томов и данных базы данных:

```bash
docker compose down -v
```

---

## Повторная сборка

В случае изменения исходного кода проекта выполните:

```bash
docker compose build
docker compose up
```

или одной командой:

```bash
docker compose up --build
```

---

## Примечания

- Проект разворачивается в полностью изолированной среде Docker.
- Для запуска не требуется установка дополнительных зависимостей на хост-системе.
- Все настройки подключения к базе данных и взаимодействия между компонентами
  заданы в `docker-compose.yml`.
